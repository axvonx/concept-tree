# Functional Programming

Functional programming (FP) treats computation as the evaluation of mathematical functions. It emphasizes immutability, pure functions (no side effects), and function composition.

## Core Concepts

- **Pure functions** — same input always gives same output, no side effects
- **Immutability** — data never mutates; transformations produce new values
- **Higher-order functions** — functions that take or return other functions (map, filter, reduce)
- **Lazy evaluation** — values computed only when needed

Languages: Haskell, Clojure, Erlang; FP features in Python, JavaScript, Scala.
